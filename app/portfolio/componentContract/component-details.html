Building a Common Component with a Wrapper

This strategy and guideline was designed as part of a larger effort to design and develop reusable common components for a large company, with thousands of developers looking to utilize the component.

My Role: I was the developer on this effort. I collaborated with the Lead Interaction Designer and Lead Visual Designer on this component to quantify UX Component Requirements into Software Requirements.

Evaluating UX Requirements

To build a common component, there must be an understanding of the expected user interaction. The designer on the project must consider potential features of the component that will be beneficial to the largest audience. Keep in mind that the component should be developed to allow for future enhancements.
