Reusable UI Components

From day to day, as an Interaction Designer we typically focus only on the end user's experience. Can users understand the intended flow? Are they able to clearly find what they're looking for? In my experience, designers will typically work with developers to ensure the prototype is achievable from a technical perspective, but when building a platform, there is actually more than one end user.

One of the more interesting projects I've been working on lately is the idea of enhancing the developer experience. When building a platform at a large corporation, there are thousands of developers building and maintaining applications, and for a large financial institution, you can't afford for the site to go down or load slowly due to inconsistent code.

The goal of building reusable UI components is to blah blah blah extending Bootstrap blah blah blah we can go from this:

Well done! You successfully read this important alert message.
Heads up! This alert needs your attention, but it's not super important.
Warning! Better check yourself, you're not looking too good.
Oh snap! Change a few things up and try submitting again.

With code that looks something like this: 

<div class="row">
    <div class="alert alert-success" role="alert">
        <strong>Well done!</strong> You successfully read this important alert message.
    </div>
</div>
<div class="row">
    <div class="alert alert-info" role="alert">
        <strong>Heads up!</strong> This alert needs your attention, but it's not super important.
    </div>
</div>
<div class="row">
    <div class="alert alert-warning" role="alert">
        <strong>Warning!</strong> Better check yourself, you're not looking too good.
    </div>
</div>
<div class="row">
    <div class="alert alert-danger" role="alert">
        <strong>Oh snap!</strong> Change a few things up and try submitting again.
    </div>
</div>

Something about sexy markup here and the idea of BEM and naming conventions so code is easier to follow.
